Angular uygulamalarınızda kullanabileceğiniz, şık ve kullanışlı bir Data Grid olan Flexi Grid ile tanışın. Bu component, OData yapısıyla tamamen uyumludur ve hem Data Binding kullanarak hem de tüm verinizi alıp işleyerek kullanabileceğiniz esnek bir yapıya sahiptir.

Flexi Grid, açık kaynak kodludur. Kaynak kodlarına buraya tıklayarak ulaşabilirsiniz.

Flexi Grid ile hem client-side hem de server-side pagination, filtreleme ve sıralama işlemlerini kolayca gerçekleştirebilirsiniz. Kullanım kolaylığı sağlamak için çok fazla emek verdim 😊. Bu component, Angular v18+ projelerinizde mükemmel bir uyum sağlar.

Projeye katkıda bulunmak ve bana destek vermek için GitHub reposuna bir yıldız (⭐) atmayı unutmayın. GitHub reposuna ulaşmak için tıklayın.

  • Data Binding özelliği ile table da yapılan her bir işlemin değerini alıp OData ile birlikte kullanabilirsiniz. Ya da istediğiniz gibi işleyebilirsiniz. Data Binding attribute true değeri verdiğinizde artık bir event ile yapılan işlemlerin çıktıları (örn pagination da sayfa değiştirme, order, filter vs) state nesnesinde alabiliyorsunuz. İsterseniz bu nesnede aldığınız değeri yine bu kütüphanede bulunan service ile otomatik OData destekli bir endpointe çevirip kullanabilirsiniz isterseniz de kendi istediğiniz gibi işleyebilirsiniz.

    Önemli not: Data Binding true yaptığınız da artık table otomatik hareket etmeyi bırakıp her şeyi sizin halletmenizi ister. Sayfa değiştirmek için sayfa numarasına bastığınızda size yeni sayfanın bilgilerini gönderir ama o sayfaya geçmez, ya da filter a değer yazdığınızda yazdığınızı state objesinde size iletir ama arama yapmaz.

Ek olarak verebileceğiniz ayarları aşağıda bulabilirsiniz

@defer { }@placeholder { }
  • height : Data grid boyutunu px olarak ayarlıyabiliyorsunuz. Default 420px
  • [autoHeight] : Data grid boyutunu otomatik ayarlasın sabit boyut kullanmasın derseniz buna true verebilirsiniz

    Not: Bu ayar true iken height verdiğiniz değer çalışmaz!

  • min-width (column) : Table'ın min-width ayarını buraya number değer vererek yapabilirsiniz. Bu ayar sayesinde o uzunluktan daha kısa sayfalarda overflow özelliği aktif olur. Yani uzunluk sabit kalır ama sayfadan taşmamak için scrolbar ile kaydırarak yan kısmı görürsünüz
  • [useMinWidth] : Min width ayarının çalışmasını istemiyorsanız bu ayarı false yapabilirsiniz. O zaman boyutunu sayfanın genişliğine göre otomatik ayarlar

    Not: Bu ayar false iken min-width için verdiğiniz değer çalışmaz!

  • [autoWidth] : Table'ın width ayarı %100 olmasın diyorsanız bu ayarı false yapıp width attributüne değer vermelisiniz

    Not: Bu ayar true iken width için verdiğiniz değer çalışmaz!

  • width (table) : Table'ın uzunluğunu bununla değiştirebilirsiniz. Default %100 dür
  • width (column) : Columnların uzunluklarını ayarlayabiliyorsunuz. Örneğin: 100px, 20% gibi
  • [visible] (column) : Eklediğiniz columnların gizli kalmasını, istediğiniz zaman görünmesini istiyorsanız bu ayar ile yapabilirsiniz
  • footerPerPageText : Pagination da bulunan page size selectinin devamındaki yazıyı isterseniz bu attribute ile değiştirebilirsiniz

autoWidth true | false false
